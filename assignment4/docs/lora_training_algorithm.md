# LoRA Training and Evaluation Contract

## Objective

Train a Stable Diffusion 1.5 style adapter for
`a busy market, in <sks> style`. The final file must contain LoRA weights for
both the UNet and text encoder plus the learned `<sks>` embedding.

## Inputs

- Base model: `runwayml/stable-diffusion-v1-5`.
- 843 supplied images under `style_imgs/512`.
- Fixed Florence-2 captions in `code/auto_captions/florence_captions.jsonl`.
- 60 images generated by unmodified SD 1.5 in `training_data/images`.
- Auxiliary provenance and sampling weights in `training_data/auxiliary.jsonl`.
- Instance token `<sks>`, which must tokenize as one new token.

No assignment-PDF image, external dataset, or adapter-generated image is an
input to the selected run.

## Outputs

- `lora_out/pytorch_lora_weights.safetensors`: one final model file.
- `training_run/run_config.json`: arguments, environment, input hashes, and
  schedule hash.
- `training_run/training_draw_schedule.jsonl`: complete source order and
  augmentation seeds.
- `training_run/training_trace.jsonl`: realized prompts, transforms, latent
  seeds, noise seeds, timesteps, and microbatch losses.
- `training_run/training_metrics.jsonl`: optimizer-step objectives and learning
  rates.
- `training_run/checkpoints/step_*`: immutable periodic adapters.
- `training_run/latest_training_state.pt`: resumable optimizer-boundary state.

## Training Algorithm

1. Validate caption coverage and load the supplied and auxiliary examples.
2. Build 2,000 weighted draws for the registered 500-step trajectory. Supplied
   images account for 86% of the effective weight; broad markets 5%; market
   character scenes 5%; and people images 4%.
3. Derive independent deterministic seeds for each draw's augmentation, VAE
   latent sample, diffusion noise, and timestep.
4. Load the SD 1.5 tokenizer, CLIP text encoder, VAE, UNet, and DDPM scheduler.
5. Add `<sks>` as one token. Initialize its row from the mean embeddings of
   `ghibli style`; keep all original embedding rows frozen.
6. Freeze the base models and attach rank-16 LoRA to UNet `to_q`, `to_k`,
   `to_v`, `to_out.0`, and rank-4 LoRA to text-encoder `q_proj`, `k_proj`,
   `v_proj`, `out_proj`.
7. Resize each image to cover 512x512, take the scheduled random crop, and apply
   the scheduled horizontal flip.
8. Use `{caption}, in <sks> style` as the styled prompt. With probability 0.08,
   use `an animated movie scene, in <sks> style`. Only truthful base-generated
   market rows may use the exact assignment prompt, according to their recorded
   per-row probability.
9. Encode the image with the frozen VAE and add scheduled diffusion noise at the
   scheduled timestep.
10. Compute Min-SNR-weighted diffusion MSE with gamma 5.
11. Disable both adapters and predict the same noisy latent from the plain
    content prompt. Penalize deviation from that frozen-base prediction with
    weight 0.65.
12. Penalize movement of the learned token from its initialization with weight
    0.05.
13. Accumulate four microbatches, clip gradients, and update AdamW. Use learning
    rates `7e-5`, `2.5e-6`, and `1e-5` for UNet, text encoder, and token;
    LoRA dropout 0.05; fp16; seed 2202; and cosine decay with 50 warmup steps.
14. Append the realized trace and metrics only after a completed optimizer step.
15. Save an immutable adapter and full resume state every 25 steps.
16. Stop at step 150, the visually selected checkpoint on the fixed 500-step
    schedule, and serialize UNet LoRA, text-encoder LoRA, and the token row into
    one canonical Safetensors file.

## Evaluation Algorithm

1. Load base SD 1.5 and replace its scheduler with DPM-Solver++.
2. Read the instance token and token embedding from the final adapter.
3. Load both LoRA branches without exposing an adapter-strength argument.
4. Render at least three 512x512 images for
   `a busy market, in <sks> style`, using consecutive deterministic seeds,
   30 inference steps, and guidance 7.5.
5. Record the adapter hash, image hashes, prompt, seeds, scheduler, dimensions,
   package versions, device, and dtype.

## Assumptions

- Full training uses a CUDA GPU with approximately 8 GB of VRAM.
- Florence captions and auxiliary images are fixed inputs; caption generation
  is not part of the required training environment.
- Different GPU architectures may reproduce the procedure and visual behavior
  without producing byte-identical floating-point weights.

## Edge Cases

- Missing, duplicate, or incomplete captions stop before model loading.
- Existing output requires `--overwrite` or an exact compatible `--resume`.
- Resume rejects a changed schedule or run contract.
- Every styled prompt contains `<sks>` exactly once; preservation prompts contain
  no custom token.
- Only LoRA parameters and the standalone token row receive gradients.
- The final output directory is cleaned to exactly one Safetensors file.
- The data verifier rejects PDF paths, the three known PDF-image hashes,
  non-SD-1.5 generators, non-null source-adapter hashes, and image hash changes.

## Expected Intermediate States

1. Data validation reports 843 supplied images and 60 auxiliary images.
2. The fixed schedule contains 2,000 draws and has a stable SHA-256.
3. Step 25 and every later 25-step boundary has an immutable adapter and resume
   state.
4. At step 150, the trace has 600 microbatch rows and metrics have 150 rows.
5. The final file has 353 tensors and includes `unet.*`, `text_encoder.*`, and
   `__custom_token_embedding__` keys.

## Verification

1. Compile the submitted Python files.
2. Validate all data paths, hashes, prompts, weights, and generator provenance.
3. Run a real CUDA smoke test and resume test.
4. Compare materialized and original auxiliary manifests; all output tensors
   and non-timing metrics must match.
5. Reproduce step 150 from `training_data` and compare every tensor with the
   submitted adapter.
6. Render and visually inspect fixed development and ten-seed holdout grids.
7. Verify the final adapter structure, sample hashes, report page count, package
   contents, ZIP CRCs, and forbidden-image hashes.
