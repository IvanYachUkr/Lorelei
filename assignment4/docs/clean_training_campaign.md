# Clean Training Campaign

## Objective

Train at least three new Stable Diffusion 1.5 LoRA adapters from the base model and
select a checkpoint that produces a clear, populated market for
`a busy market, in <sks> style` while retaining coherent people, visible faces,
the supplied visual style, and prompt control.

Visual checkpoint review decides model selection. Training loss is diagnostic only.

## Assignment Contract

- Base model: `runwayml/stable-diffusion-v1-5`.
- Add `<sks>` as exactly one tokenizer token.
- Train LoRA adapters on both the UNet and text encoder.
- UNet targets: `to_q`, `to_k`, `to_v`, and `to_out.0`.
- Text encoder targets: `q_proj`, `k_proj`, `v_proj`, and `out_proj`.
- Save the learned token embedding with both adapters in one Safetensors file.
- The final evaluation uses the exact assignment prompt, renders at least three images,
  and exposes no adapter-strength parameter.
- The final training script accepts the required data directory, token, output directory,
  hyperparameters, and `--overwrite` arguments.

## Allowed Inputs

1. The 843 images in `style_imgs/512` and their Florence-2 captions.
2. Deterministic crops, flips, and other transforms derived from those images.
3. Face/person detections and crops produced from those images with Florence-2.
4. Images generated by the unmodified SD 1.5 base model from recorded prompts and seeds.

Images embedded in the assignment PDF, external datasets, and images produced by other
generative models are forbidden. Adapters predating this clean campaign are evaluation
references only and are never initialization or data generators.

The provenance validator rejects:

- any path containing `market_refs` or identifying a PDF extraction;
- the three known SHA-256 hashes of the PDF-embedded market images;
- a derived crop whose source is outside `style_imgs/512`;
- a generated image whose manifest does not identify the unmodified SD 1.5 base model,
  a prompt, seed, scheduler, step count, guidance value, and image hash;
- a generated image with a non-null source-adapter hash.

## Shared Training Method

1. Build a complete weighted draw schedule before loading the model.
2. Derive crop, flip, prompt, VAE, diffusion-noise, and timestep seeds from the run seed
   and draw index. Log every realized decision.
3. Resize to 512, use deterministic random crops and horizontal flips, and avoid color
   jitter so the supplied palette is not diluted.
4. Form style prompts as `{caption}, in <sks> style`; use a generic animated-scene prompt
   on a small caption-dropout fraction. Only truthful base-generated market examples may
   receive the exact assignment prompt.
5. Initialize the custom token from the mean embedding of `ghibli style`.
6. Train rank-16 UNet LoRA and rank-4 text-encoder LoRA with FP16, batch 1, four-step
   gradient accumulation, gradient checkpointing, Min-SNR gamma 5, model-preservation
   loss, token anchoring, AdamW, and cosine decay.
7. Save immutable adapters and resumable optimizer state every 25 optimizer steps.
8. Run training in 100-step sessions against one fixed full-run schedule. After every
   session, render all new 25-step checkpoints with fixed seeds and inspect the actual
   images before resuming. This is substantially more frequent than a ten-epoch review:
   25 optimizer steps represent only 100 image draws, about 0.12 of the supplied set.

## Candidate A: Supplied Only

- Inputs: 843 supplied images only.
- Steps: up to 500; seed 1101.
- Learning rates: UNet `8e-5`, text encoder `2.5e-6`, token `1.2e-5`.
- Caption dropout: 0.10; preservation: 0.60; token anchor: 0.05.
- Purpose: recover the strong character and face behavior of the earliest model while
  measuring how much market semantics survive without target-specific images.

## Candidate B: Self-Market

- Inputs: supplied images plus curated images generated by unmodified SD 1.5.
- Effective draw fractions: 86% supplied, 5% broad markets, 5% medium-shot market
  characters, and 4% people/portraits.
- Exact assignment prompt probability: 0.25 for broad markets and 0.35 for medium-shot
  market-character images; zero for portraits.
- Steps: up to 500; seed 2202.
- Learning rates: UNet `7e-5`, text encoder `2.5e-6`, token `1.0e-5`.
- Caption dropout: 0.08; preservation: 0.65; token anchor: 0.05.
- Purpose: restore market composition and foreground people without relying on any
  external image source.

## Candidate C: Balanced People

- Inputs: supplied images, the same SD 1.5 self-samples at a lower combined fraction,
  and deterministic face/person crops derived from supplied images.
- Effective draw fractions: 82% supplied, 4% broad markets, 4% medium-shot market
  characters, 2% base-model people/portraits, and 8% supplied-image crops.
- No spatial mask weighting is used.
- Steps: up to 600; seed 3303.
- Learning rates: UNet `6e-5`, text encoder `2.0e-6`, token `8e-6`.
- Caption dropout: 0.08; preservation: 0.70; token anchor: 0.06.
- Purpose: test whether a restrained amount of real style-consistent face scale improves
  people while avoiding the produce-heavy failure of the earlier face-mask run.

## Candidate D: Market-to-People Refinement

The first three candidates showed that Candidate B step 150 retained the strongest
market structure, while Candidate C step 150 improved close and medium faces but did not
consistently improve small crowd faces. Candidate D is therefore a short continuation of
the eligible, fully logged Candidate B step-150 adapter using Candidate C's clean balanced
manifest.

- Initialization: Candidate B step 150, including its learned `<sks>` embedding.
- Inputs: the verified `balanced_people.jsonl` mixture; no new images.
- Steps: 60 new optimizer steps; seed 4404.
- Learning rates: UNet `1.5e-5`, text encoder `5e-7`, token `1e-6`.
- Scheduler: cosine with 5 warmup steps.
- Caption dropout: 0.03; preservation: 0.75; token anchor: 0.10.
- Checkpoints and visual review: steps 15, 30, 45, and 60.
- Purpose: add a restrained face/person signal to B's market-capable adapter without
  flattening or washing out its scene geometry.

The initialization path and SHA-256 are part of the immutable run contract. This is a
separate optimization trajectory with a fresh optimizer, scheduler, draw schedule, and
RNG seed; it is not represented as a fresh-from-base candidate.

## Visual Review Protocol

At steps 25, 50, 75, 100, and every following 25-step checkpoint, render fixed seeds for:

- the exact assignment prompt;
- a market vendor facing the viewer;
- two shoppers talking beside a vegetable stall;
- a family choosing food at a busy market;
- a close portrait of a smiling market vendor;
- a non-market scene using `<sks>`; and
- `a busy market` without `<sks>` as the token-control check.

Each review records concise observations for market readability, composition, people and
face coherence, style fidelity, brightness/color stability, and artifacts. A checkpoint is
accepted only when it has a clear market in at least five of six exact-prompt seeds, people
in at least five, no severe scene collapse, and visibly stronger overall quality than the
supplied-only candidate. A second candidate is retained to demonstrate the effect of the
training-data change even if it is not selected.

## Outputs

- Three independent clean-from-base adapters plus the evidence-driven refinement.
- Per-run configuration, input hashes, complete draw schedule, microbatch trace, metrics,
  immutable checkpoints, resume state, console log, and visual-review images.
- An asset registry containing every allowed source, transform, prompt, seed, and hash.
- A written visual review and final selection record.
- A final one-file dual-adapter/token checkpoint, deterministic samples, README, report,
  requirements, and validated archive after selection.

## Edge Cases

- OOM: keep 512 resolution and LoRA ranks fixed; reduce only evaluation batch size.
- Interruption: resume only from an optimizer-boundary state with matching config and
  schedule hashes.
- Loss improves while images regress: retain the earlier visual checkpoint.
- Faces improve but market structure fails: do not select the model for the assignment.
- Market structure improves but people disappear: reduce target/self-sample share in the
  contingency run rather than adding spatial face loss.
- Any forbidden hash or unresolved provenance: reject the entire run before training.

## Expected Intermediate States

1. Provenance gate passes with 843 supplied images, zero forbidden hashes, and fixed
   auxiliary manifests.
2. A two-step smoke run produces one final adapter, one periodic checkpoint, state,
   schedule, trace, metrics, and deterministic evaluation images.
3. Each 100-step session ends at the requested boundary with an immutable checkpoint and
   resumable state.
4. Each visual review is written before that candidate is resumed or the next candidate
   begins.
5. Final selection names a specific candidate and optimizer step, never merely a run
   endpoint or lowest-loss checkpoint.
